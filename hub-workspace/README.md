# Hub Workspace — Multi-Model Agent Runtime

Hub Workspace is the chat surface inside the **Limitless Stack Hub** (route `/chat`, sidebar entry "Hub Workspace" under the Personal section). It's where humans and agents converse, and the runtime where Limitless Stack agents are dispatched.

## Role in the Stack

Hub Workspace is the orchestration and execution layer. It provides:

- **Multi-model dispatch** — Gemini 2.5 Flash by default (free for onboarded users), Claude as opt-in for complex multi-file reasoning. Switchable per-conversation.
- **Presence-aware** — The Hub Workspace tile on `/today` glows when a teammate is mid-conversation. Workspace-scoped so every member sees who's active.
- **Workspace integration** — The vault, repos, Pinecone search, NotebookLM panel, and Paperclip approvals are all accessible from the same Hub UI.
- **Skill system** — Agents dispatched from Hub Workspace can install skills (including this one) to learn protocols.

For a 100-vertical platform, Hub Workspace is how you parallelize development work — multiple agents working on different verticals simultaneously, each following the Limitless Stack protocol, all coordinated through one shared UI.

## Role in Self-Healing

Hub Workspace enables key self-healing capabilities:

- **Parallel fix candidates** — For high-severity bugs, the Hub can dispatch multiple agent runs with varied approaches, producing multiple PRs for operators to compare. Cost is bounded (3x a single attempt) and gives operators better fix options.
- **Cross-model verification** — A fix generated by Claude can be reviewed by a Gemini agent for a second opinion, reducing false positives.
- **Bulk rollout** — When shipping the self-healing standard to new apps, Hub Workspace can run parallel agents that each scaffold the canonical files into a different repo.

## Setup

Hub Workspace ships as part of the Limitless Stack Hub (`Open-Scaffold-Labs/limitless-stack-hub` on GitHub, deployed to `https://limitless-stack-hub.vercel.app`). Sign in with GitHub OAuth (gated on `Open-Scaffold-Labs` org membership), open the Personal → Hub Workspace tab, and pick a model.

## Models available

| Model | Best for |
|---|---|
| Gemini 2.5 Flash (default) | General tasks, fast iteration, free-tier for onboarded users |
| Claude Opus 4.6 | Complex multi-file reasoning, architecture decisions |
| Claude Sonnet 4.6 | Balanced speed and capability, diagnostic passes |
| Claude Haiku 4.5 | Lightweight calls, batch tooling |

## Integration points

- **Claude / Gemini** — Hub Workspace dispatches either as the reasoning engine, switchable per conversation
- **CLAUDE.md** — Each agent reads the relevant CLAUDE.md (vault or per-repo) for context
- **Obsidian** — The vault is browsable from the Hub `/wiki` page; Hub Workspace agents can read and update it
- **Pinecone** — Agents search the corpus via the Hub `/pinecone` page or via the underlying scripts
- **NotebookLM** — Agents query topic notebooks via the Hub `/notebooks` directory or the `notebooklm` CLI on the host machine
- **Paperclip** — Agents report task completion to Paperclip; approvals flow through `/approvals`
- **Self-healing** — Parallel fix candidates, cross-model verification, bulk rollout of self-heal standard
